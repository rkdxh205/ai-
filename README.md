# AI 활용 업무 혁신 해커톤 심사 시스템

> 이노커브AI가 운영하는 해커톤 전용 AI 심사 도구입니다.  
> 활동계획서(기획)와 앱 소스코드를 업로드하면 GPT-4o가 5개 영역·11개 세부 항목·100점 만점으로 자동 채점합니다.

## 주요 기능

| 기능 | 설명 |
|------|------|
| **파일 업로드** | 활동계획서(docx/txt/md), 소스코드(py/json/yaml 등), 추가 파일 최대 3종 동시 업로드 |
| **AI 자동 채점** | GPT-4o 기반 스트리밍 심사 — 결과가 실시간으로 화면에 출력됨 |
| **점수 시각화** | 최종 총점을 히어로 카드로 즉시 표시 |
| **PDF 다운로드** | 심사 결과를 한글 폰트(나눔고딕) 적용 PDF로 저장 |
| **세션 유지** | 페이지 새로고침 없이 이전 심사 결과가 유지됨 |

## 기술 스택 및 패키지

### 언어 및 프레임워크

| 항목 | 버전 | 역할 |
|------|------|------|
| Python | 3.9+ | 애플리케이션 런타임 |
| Streamlit | `>=1.35.0` | 웹 UI 프레임워크. 파일 업로더·버튼·탭·스피너 등 모든 UI 컴포넌트 제공 |

### 핵심 라이브러리

| 패키지 | 버전 | 용도 |
|--------|------|------|
| `openai` | `>=1.30.0` | GPT-4o API 호출. `chat.completions.create(stream=True)`로 스트리밍 심사 수행 |
| `python-dotenv` | `>=1.0.0` | `.env` 파일에서 `OPENAI_API_KEY` 자동 로드 |
| `python-docx` | `>=1.1.0` | `.docx` 형식의 활동계획서를 파싱해 텍스트 추출 |
| `fpdf2` | `>=2.7.0` | 마크다운 심사 결과를 한글 PDF로 변환. 나눔고딕 TTF 폰트 임베드 |

### 표준 라이브러리 (별도 설치 불필요)

| 모듈 | 용도 |
|------|------|
| `re` | 최종 총점 숫자 추출, 마크다운 인라인 서식 제거 |
| `io` | `BytesIO`로 업로드된 파일을 메모리에서 직접 처리 |
| `base64` | 로고 이미지를 HTML `data:image/png;base64,...` 형식으로 인코딩 |
| `os` | 환경변수 읽기, 폰트 파일 경로 조합 |
| `datetime` | PDF 파일명에 타임스탬프(YYYYMMDD_HHMMSS) 생성 |

## 프로젝트 구조

```
hackathon_judge/
├── app.py                    # 메인 애플리케이션 (단일 파일)
├── requirements.txt          # Python 패키지 의존성 목록
├── .env                      # 환경변수 (OPENAI_API_KEY) — git 제외
├── .gitignore                # git 추적 제외 파일 설정
├── 이노커브AI 로고.png         # 히어로 섹션 로고 이미지
└── fonts/
    ├── NanumGothic-Regular.ttf   # PDF 본문용 한글 폰트
    └── NanumGothic-Bold.ttf      # PDF 제목·강조용 한글 폰트
```

### `app.py` 내부 구성

```
app.py
├── CSS 스타일 정의       (st.markdown 글로벌 커스텀 CSS)
├── SYSTEM_PROMPT         (GPT-4o에 전달하는 심사 지침 및 루브릭)
│
├── read_uploaded_file()  파일 파싱 함수 (docx / 텍스트 인코딩 자동 감지)
├── build_user_message()  활동계획서·코드·추가파일 → GPT 입력 메시지 조합
├── generate_pdf()        마크다운 심사 결과 → fpdf2 PDF 변환
├── run_evaluation()      OpenAI API 스트리밍 호출 → chunk yield
│
└── UI 렌더링
    ├── 히어로 섹션        (로고 + 제목 + 통계 배지)
    ├── 채점 기준 그리드   (5개 카드 + 주요 감점 포인트)
    ├── 업로드 영역        (3컬럼 — 활동계획서 / 소스코드 / 추가파일)
    ├── 심사 시작 버튼     (유효성 검사 → run_evaluation 호출)
    └── 결과 표시 영역     (점수 히어로 카드 + 마크다운 결과 + PDF 다운로드)
```

## 실행 프로세스

```
[사용자] 파일 업로드
        ↓
활동계획서 파싱 (read_uploaded_file)
  • .docx → python-docx로 단락 추출
  • .txt/.md → UTF-8/CP949/EUC-KR 인코딩 자동 감지
        ↓
소스코드 + 추가 파일 파싱 (동일 방식)
        ↓
GPT 입력 메시지 조합 (build_user_message)
  • "활동계획서 및 기획 문서" 섹션
  • "앱 소스코드" 섹션 (코드블록 포함)
  • "추가 제출 파일" 섹션 (선택)
        ↓
OpenAI GPT-4o API 호출 (run_evaluation)
  • model: gpt-4o
  • max_tokens: 8096
  • temperature: 0  (일관성 보장)
  • seed: 42        (재현성 보장)
  • stream: True    (청크 단위 실시간 출력)
        ↓
스트리밍 결과 화면 출력 (result_placeholder)
  • 청크 누적 → 마크다운 실시간 렌더링
        ↓
정규식으로 총점 숫자 추출 → 히어로 카드 업데이트
        ↓
마크다운 → HTML 후처리 (h2/h3/h4/li 태그 스타일 적용)
        ↓
generate_pdf() — fpdf2로 PDF 생성
  • 나눔고딕 TTF 폰트 로드
  • 마크다운 줄별 파싱 → 제목/목록/구분선 렌더링
  • 이모지 → ASCII 대체 ([O]/[X])
        ↓
[사용자] PDF 다운로드 버튼 클릭
```

## 설치 및 실행 방법

### 1. 저장소 클론

```bash
git clone <저장소 URL>
cd hackathon_judge
```

### 2. 가상환경 생성 (권장)

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 3. 의존성 설치

```bash
pip install -r requirements.txt
```

### 4. 환경변수 설정

```bash
# .env 파일 생성
echo OPENAI_API_KEY=sk-xxxx > .env
```

### 5. 앱 실행

```bash
streamlit run app.py
```

브라우저에서 `http://localhost:8501` 접속

## 사용 방법

1. **활동계획서 업로드** — 기획 문서를 `.docx`, `.txt`, `.md` 형식으로 업로드합니다.
2. **소스코드 업로드** — 구현된 Python 스크립트, 설정 파일 등을 업로드합니다. (`.py`, `.json`, `.yaml`, `.toml` 지원)
3. **추가 파일 업로드** *(선택)* — 프롬프트 파일, README 등 보조 문서를 추가합니다.
4. **심사 시작** 버튼을 클릭합니다.
5. GPT-4o가 심사를 진행하며 결과가 실시간으로 화면에 출력됩니다.
6. 심사 완료 후 **PDF 다운로드** 버튼으로 결과를 저장합니다.

## 주의사항

- GPT-4o 호출 비용이 발생합니다. 제출물 분량에 따라 입력 토큰이 많아질 수 있습니다.
- 소스코드가 길 경우 `max_tokens: 8096` 한도 내에서 심사 결과가 생성됩니다.
- PDF 생성 시 이모지는 ASCII 대체 문자(`[O]`, `[X]`)로 변환됩니다.
- 폰트 파일(`fonts/NanumGothic-*.ttf`)이 없으면 PDF 생성이 실패합니다.
